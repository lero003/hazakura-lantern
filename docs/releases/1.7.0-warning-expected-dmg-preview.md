## Proposed Release Note Addendum

This is a warning-expected DMG preview. It is not the source-only checkpoint
release.

The preview is for users who want to inspect the locally built app without
running the build steps themselves.

The DMG is a developer-preview convenience artifact. It is ad-hoc signed only.
It is not Developer ID signed, not notarized, and macOS may show security
warnings or reject normal Gatekeeper assessment.

Only use this preview if you are comfortable evaluating unsigned and
not-notarized macOS preview builds.

Artifact:

```txt
hazakura-lantern_1.7.0_aarch64-warning-expected.dmg
```

Platform:

```txt
macOS Apple Silicon / aarch64
```

SHA-256:

```txt
0951a3b8c9f544d2e8e06c503e51301b91c6d4bccb09aabe9811721db48afb96
```

Local verification completed before publication approval:

```bash
git diff --check
swift test
swift build --disable-sandbox
script/build_and_run.sh --verify
script/build-warning-dmg-preview.sh
hdiutil verify dist/dmg/hazakura-lantern_1.7.0_aarch64-warning-expected.dmg
shasum -c dist/dmg/hazakura-lantern_1.7.0_aarch64-warning-expected.dmg.sha256
```

The DMG mounted locally. The contained `Hazakura Lantern.app` matched the
source app bundle id, version, build, and CDHash. `spctl` rejected the app,
which matches the warning-expected, not-notarized preview boundary.

## Required Publication Caveat

Do not describe this artifact as signed for public distribution, notarized,
trusted, installer-grade, stable, or production-ready.

Do not attach this DMG to a release until the user explicitly approves binary
asset publication.

## Known Packaging Note

The warning-expected preview lane uses `script/build-warning-dmg-preview.sh`,
which creates a plain DMG directly with `hdiutil create` from the freshly built
app bundle. Treat this as acceptable only for the warning-expected preview lane.
A Developer ID / notarized DMG lane needs separate signing, hardened runtime,
notarization, stapling, and Gatekeeper verification work.
